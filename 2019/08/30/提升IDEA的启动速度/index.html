提升IDEA的启动速度

发表于 2019-08-30 评论数：

症状与原因

在Windows 10安装了IDEA后，每次启动光是单击到出现界面都耗时超过30秒，我的电脑配置也并不低，250G SSD+i7-7700+16G内存，即使是Visual Studio这种重量级选手也是秒开的，按理说IDEA不应该这么慢。

遂google搜了一圈，发现Windows Defender是罪魁祸首。每次IDEA启动的时候，Windows Defender的实时监控程序会对exe进行扫描，扫描耗时很长。

想象一下，当你充满激情想写两行代码时，点击IDEA图标，发现半天没动静，于是你去倒了杯水，上了个厕所回来发现还没起来，你拿出手机，刷了下微信微博，两个小时过去了，才想起来自己是要写代码来着，你怒火中烧：垃圾IDEA，毁我青春。

解决方案

打开 设置> 更新与安全> Windows安全中心> 病毒和威胁防护> 管理设置> 添加或删除排除项。把idea64.exe添加到进程排除项中即可。

其他基于IntelliJ平台的IDEs进程名称：

  • webstorm64.exe
  • datagrip64.exe
  • clion64.exe
  • rider64.exe
  • datagrip64.exe
  • goland64.exe
  • phpstorm64.exe
  • pycharm64.exe
  • rubymine64.exe
  • webstorm64.exe
  • studio64.exe

常见的开发工具Windows Defender排除项参考这份脚本清单 >> Windows Defender Exclusions

其他原因

还有一个导致IDEA启动慢的原因是插件太多，默认的插件很多是用不上的，导致启动时加载插件慢，可以根据自己的开发需求选择禁用一些插件。

解决办法：Settings>Plugins>Installed 界面中禁用一些自己用不上的插件，比如安卓插件、JSP插件等…
